# 系统架构设计文档

## 系统概述

轮胎AI模式识别系统采用模块化分层架构，实现从图像输入到质量评分的端到端自动化流程。

### 核心技术栈

- **编程语言**：Python 3.12
- **图像处理**：OpenCV, Pillow
- **数据处理**：NumPy
- **测试框架**：Pytest
- **日志系统**：Python logging

## 整体架构

```
┌─────────────────────────────────────────┐
│         应用层 (Application)          │
├─────────────────────────────────────────┤
│         业务层 (Business)              │
├─────────────────────────────────────────┤
│         服务层 (Service)               │
│  Preprocessor | Inference | Postproc  │
│  Analyzers   | Scorer                  │
├─────────────────────────────────────────┤
│         工具层 (Utility)               │
│  Logger | Exceptions | CV Utils | IO  │
├─────────────────────────────────────────┤
│         配置层 (Config)                │
├─────────────────────────────────────────┤
│         数据层 (Data)                  │
└─────────────────────────────────────────┘
```

## 核心模块

### 1. 预处理模块 (Preprocessor)

- 图像格式验证
- CMYK颜色空间转换
- 灰边去除
- 图像尺寸标准化
- 质量检查

### 2. 推理模块 (Inference)

- 轮胎花纹识别
- 中部区域花纹生成
- 边缘区域花纹生成
- 模式特征提取

### 3. 后处理模块 (Postprocessor)

- 小图筛选
- 纵图拼接
- 横图拼接
- 结果整合

### 4. 分析器模块 (Analyzers)

- 图案连续性检测（支持方法A和B）
- 图像质量分析
- 模式特征分析

### 5. 评分模块 (Scorer)

- 业务规则评分
- 美感质量评分
- 综合评分计算

### 6. 工具模块 (Utils)

- **日志系统**：文件和控制台双输出，可配置日志级别
- **异常处理**：完整的异常层次结构（TireAIError基类）
- **图像处理**：图像读取/保存、格式转换、质量检查
- **IO工具**：文件读写、路径处理

## 数据结构

### 主要数据类

- **Task**: 任务数据结构
- **ImageData**: 图像数据结构
- **DetectionResult**: 检测结果数据结构
- **ContinuityResult**: 连续性检测结果

## 关键技术

### 1. 连续性检测算法

基于图像边缘端点的匹配算法：
- 自适应阈值计算（Otsu算法）
- 两种端点提取方法（纯像素操作/轮廓检测）
- 粗细线识别和匹配
- 匹配率计算和判定

### 2. 图像拼接技术

- 基于连续性检测的智能拼接
- 自动对齐算法
- 重叠区域优化

### 3. 性能优化

- NumPy向量化操作
- 内存预分配
- 批处理优化
- 并发处理支持

### 4. 异常处理机制

- 分层异常处理
- 自定义异常体系
- 错误恢复机制

## 数据流向

```
输入图像 → 预处理 → 推理 → 后处理 → 评分 → 输出结果
                  ↑
              分析器
```
