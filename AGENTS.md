# AGENTS.md

本文件用于告诉 AI 在不同任务场景下优先查阅哪些项目文档和协作规范。

## 通用入口

- 需要快速了解项目背景、模块入口和上下文时，阅读：
  - `docs/ai_context_entrypoint.md`

## 编码规范

- 修改或新增代码前，需要了解项目编码约定时，阅读：
  - `docs/project_coding_standards.md`

- 需要了解 AI 协作时的额外编码要求时，阅读：
  - `docs/project_coding_standards_agent.md`

## 提交与 PR

- 需要生成 commit message、拆分提交或判断提交粒度时，阅读：
  - `.github/commit_conventions.md`

- 需要创建 PR、生成 PR 标题/正文或确认 PR 结构时，使用：
  - `.github/pull_request_template.md`

## 异常处理

- 需要设计、修改或使用项目异常类型时，阅读：
  - `docs/exception_handling_design.md`
  - `docs/exceptions_usage.md`

## 日志

- 需要新增日志、调整日志格式或确认日志使用方式时，阅读：
  - `docs/logger_usage.md`

## 数据模型

- 需要修改或使用 Pydantic 模型、图片模型、规则模型时，阅读：
  - `docs/models_usage.md`

## 架构说明

- 需要了解系统整体架构、模块边界或数据流时，阅读：
  - `ARCHITECTURE.md`

## 方案与计划

- 需要查看历史实现方案、设计记录或阶段计划时，阅读：
  - `docs/plans/`
