### 文档 2：`ARCHITECTURE.md` 更新

**更新后的架构图：**
```
┌─────────────────────────────────────────┐
│         服务层 (Service)                │
│  Preprocessor | Inference | Postproc   │
│  Scorer                                 │
├─────────────────────────────────────────┤
│         规则层 (Rules)                  │
│  rule1to5 | rule6_1 | rule6_2 | ...    │
├─────────────────────────────────────────┤
│         算法层 (Algorithms)             │
│  detection/     |    stitching/         │
│  (连续性检测)   |    (纵图/横图拼接)    │
├─────────────────────────────────────────┤
│         工具层 (Utility)                │
│  Logger | Exceptions | CV Utils | IO   │
├─────────────────────────────────────────┤
│         配置层 (Config)                 │
│  base | module | postprocessor | rules │
└─────────────────────────────────────────┘
```

**数据流向：**
```
输入图像 → 预处理 → 推理* → 后处理 → 评分 → 输出结果
                         ↓        ↓
                       rules ←────┘
                         ↓
                    algorithms
                         ↓
                      工具层

* 推理：外部模型调用，模型输出存储到约定位置
```

**需要更新的部分：**

| 章节 | 变更内容 |

### 文档 4：`docs/api_postprocessor.md` 新增

**文档结构：**
```markdown
# 后处理 API 参考文档

## 1. 概述

后处理模块提供两种使用方式：
1. 命令行脚本 - 适合独立运行
2. 服务类 API - 适合集成调用

## 2. 命令行脚本 API

### scripts/postprocessor.py

#### 参数说明
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| --task-id | str | 是 | 任务 ID |
| --user-conf | str | 否 | 用户配置文件路径 |

#### 使用示例
```bash
# 基本用法
python scripts/postprocessor.py --task-id 1778457600

# 带用户配置
python scripts/postprocessor.py --task-id 1778457600 --user-conf /path/to/config.json
```

#### 输出说明
- 成功：返回 tuple (True, result_dict)
- 失败：返回 tuple (False, {"failed_stage": "...", "err_msg": "..."})

#### 常见使用场景

**场景 1：标准流程运行**
```bash
# 步骤 1: 准备数据
python scripts/pieces_2_standard_input.py --task_id 1778457600

# 步骤 2: 运行后处理
python scripts/postprocessor.py --task-id 1778457600

# 步骤 3: 查看结果
ls .results/tasks/1778457600/combine/
```

**场景 2：使用自定义配置**
```json
// my_config.json
{
  "vertical_stitch_conf": {
    "enabled": true
  },
  "decoration_conf": {
    "border_width": 10
  }
}
```
```bash
python scripts/postprocessor.py --task-id 1778457600 --user-conf my_config.json
```

**场景 3：集成到 Python 代码**
```python
from services.postprocessor import postprocessor

# 方式 1: 直接调用
success, result = postprocessor(
    task_id="1778457600",
    user_conf={}  # 或使用配置字典
)

if success:
    print(f"生成图片数量：{result['image_gen_number']}")
    for idx, img_info in result.items():
        if idx.isdigit():
            print(f"图片{idx}: 得分={img_info['image_score']}")
else:
    print(f"失败阶段：{result['failed_stage']}")
    print(f"错误信息：{result['err_msg']}")
```

## 3. 服务类 API

### services/postprocessor.py

#### 函数签名
```python
def postprocessor(task_id: str, user_conf: dict | str) -> tuple[bool, dict]
```

#### 参数说明
| 参数 | 类型 | 说明 |
|------|------|------|
| task_id | str | 任务 ID，用于定位输入和输出目录 |
| user_conf | dict 或 str | 用户配置（字典或配置文件路径） |

#### 返回值说明
```python
# 成功
(True, {
    "image_gen_number": 3,
    "0": {"image_score": 85, ...},
    "1": {"image_score": 90, ...},
    ...
})

# 失败
(False, {
    "failed_stage": "vertical_stitch",
    "err_msg": "错误信息"
})
```

#### 9 阶段流程
后处理按顺序执行以下 9 个阶段：
1. Conf 处理
2. 小图筛选
3. 小图打分
4. 纵图拼接
5. 横图拼接
6. 横图打分
7. 装饰边框
8. 统计总分
9. 整理输出

#### 错误处理
- 任何阶段失败会立即返回，不继续执行后续阶段
- 错误信息包含失败阶段名称和具体错误原因

## 4. 配置项说明

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| small_image_filter_conf | 小图筛选配置 | ... |
| vertical_stitch_conf | 纵图拼接配置 | ... |
| horizontal_stitch_conf | 横图拼接配置 | ... |
| decoration_conf | 装饰边框配置 | ... |

```markdown
# 配置系统指南

## 配置目录结构

```
configs/
├── __init__.py
├── base_config.py      # 基础系统配置
├── complete_config.py  # 完整配置（聚合所有子配置）
├── module_config.py    # 模块配置
├── postprocessor_config.py  # 后处理配置
├── rules_config.py     # 规则配置
└── user_config.py      # 用户自定义配置
```

## 各配置文件说明

### base_config.py
- 系统路径配置
- 日志配置
- 默认参数

### postprocessor_config.py
- 9 阶段流程的各阶段配置
- 参数说明表格

### rules_config.py
- 规则编号与配置映射
- 规则参数说明

### user_config.py
- 用户自定义配置项
- 覆盖优先级说明

## 配置使用方式

### 方式 1: 代码中引用
```python
from configs.postprocessor_config import PostprocessorConfig

config = PostprocessorConfig()
```

### 方式 2: 通过 CompleteConfig 获取
```python
from configs.complete_config import CompleteConfig

cc = CompleteConfig()
conf = cc.to_legacy_dict()
```

### 方式 3: 用户配置覆盖
```python
user_conf = {
    "vertical_stitch_conf": {
        "enabled": True
    }
}
```

## 配置优先级

```
用户配置 > 子配置 > 基础配置
```
```

**注意**: 配置项默认值需要从配置文件源码中提取。
