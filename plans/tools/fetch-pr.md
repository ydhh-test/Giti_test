# PR 拉取脚本计划

## 脚本基本信息

- **名称**: `tools/fetch-pr.sh`
- **语言**: Bash（兼容 zsh）
- **用途**: 从 GitHub 拉取 PR 代码到本地，基于最新主分支创建审查环境

---

## 命令行接口设计

```bash
# 拉取 PR 代码
./tools/fetch-pr.sh --pr 42
./tools/fetch-pr.sh --pr 42 --branch main

# 只更新主分支（不拉取 PR）
./tools/fetch-pr.sh --update-only
./tools/fetch-pr.sh --update-only --branch main

# 显示帮助
./tools/fetch-pr.sh --help

# 详细输出/静默模式
./tools/fetch-pr.sh --pr 42 --verbose
./tools/fetch-pr.sh --pr 42 --quiet
```

| 参数 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `--pr` | 条件必填 | 无 | GitHub PR 编号（数字），与 `--update-only` 互斥 |
| `--update-only` | 条件必填 | false | 只更新主分支，不拉取 PR，与 `--pr` 互斥 |
| `--branch` | 否 | `dev` | 主分支名称 |
| `--verbose` | 否 | false | 详细输出 |
| `--quiet` | 否 | false | 静默模式 |
| `--help` | 否 | - | 显示帮助 |

---

## 参数验证规则

| 输入组合 | 验证结果 |
|----------|----------|
| `--pr 42` | ✅ 有效 |
| `--pr dev` | ❌ 报错：PR 编号必须是数字 |
| `--update-only` | ✅ 有效 |
| `--pr 42 --update-only` | ❌ 报错：参数互斥 |
| 无参数 | ❌ 报错：缺少必填参数 |

## 完整执行流程

```
┌─────────────────────────────────────────────────────────┐
│  1. 解析参数                                              │
│     - 验证 --pr 和 --update-only 互斥                     │
│     - 验证 --pr 是数字                                    │
│     - 验证 --branch 非空                                  │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  2. 检查是否在 git 仓库内                                  │
│     - 不在 → 报错退出                                    │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  3. 检查当前是否在指定的主分支上                           │
│     - 不在 → 报错退出（提示用户先 checkout 到主分支）        │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  4. 检查是否有未提交的更改                                 │
│     - 有 → 报错退出                                      │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  5. git fetch origin <主分支>                             │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  6. git pull --rebase origin <主分支>                     │
│     (已在主分支，无需 checkout)                            │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  7. 递归清理缓存文件                                       │
│     (.logs, .results, .pytest_cache, __pycache__)        │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  判断：是否是 --update-only 模式？                         │
│     - 是 → 跳到步骤 10（完成）                             │
│     - 否 → 继续步骤 8                                     │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  8. 检查 pr-review/<PR编号> 分支是否存在                   │
│     - 存在 → 报错退出，让用户手动删除                       │
│     - 不存在 → 继续                                      │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  9. git fetch origin pull/<PR编号>/head:pr-review/<PR编号>│
│     git checkout pr-review/<PR编号>                       │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  10. 输出完成信息，显示当前分支状态                         │
└─────────────────────────────────────────────────────────┘
```

## 错误类型与处理

| 错误类型 | 错误信息示例 | 是否回滚 |
|----------|--------------|----------|
| 参数错误 | "错误：--pr 和 --update-only 不能同时使用" | 否 |
| PR 编号非数字 | "错误：PR 编号必须是数字，收到：dev" | 否 |
| 不在 git 仓库 | "错误：当前目录不是 git 仓库" | 否 |
| 不在主分支 | "错误：请先切换到 dev 分支后运行此脚本" | 否 |
| 有未提交更改 | "错误：请先提交或暂存当前更改" | 否 |
| PR 分支已存在 | "错误：分支 pr-review/42 已存在，请先删除" | 否 |
| 网络错误 | "错误：无法连接到远程仓库" | 是（删除部分创建的分支） |
| PR 编号无效 | "错误：PR #42 不存在" | 是 |

## 新分支命名方案

**格式**: `pr-review/<PR 编号>`

例如：`pr-review/42`

**理由**:
- `pr-review/` 前缀清晰表明用途
- 与功能分支命名区分开
- 便于后续批量清理

**分支已存在处理**: 报错退出，让用户手动删除

---

## 补充说明

1. **--update-only 模式**: 当用户只想更新主分支而不拉取 PR 时使用，此时不需要传入 `--pr` 参数

2. **PR 编号验证**: PR 编号必须是纯数字，传入 `dev` 等非数字值时报错

3. **执行顺序**: 先检查是否在主分支，再执行 fetch 动作，因此无需 checkout 到主分支

4. **清理范围**: 递归清理全仓库，包括所有子目录下的 `__pycache__`

5. **错误回滚**: 对于可逆操作（如创建分支），失败时自动回滚；对于不可逆操作（如清理文件），记录日志供参考
